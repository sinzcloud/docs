# 最佳落地建议（避免后期灾难）

## 1）服务端/客户端共用同一份常量文件

* Java：`Cmd.java`
* TS：`Cmd.ts`

## 2）严格规定：请求 cmd 和推送 cmd 分开

例如：

* 请求：1401 ROOM_JOIN
* 推送：9101 PUSH_ROOM_JOINED

避免 cmd 混乱。

## 3）所有游戏都必须有 Snapshot cmd

断线重连必须要快照，不然必崩。

# 最佳落地建议（避免后期灾难）

## 1）服务端/客户端共用同一份常量文件

* Java：`Cmd.java`
* TS：`Cmd.ts`

## 2）严格规定：请求 cmd 和推送 cmd 分开

例如：

* 请求：1401 ROOM_JOIN
* 推送：9101 PUSH_ROOM_JOINED

避免 cmd 混乱。

## 3）所有游戏都必须有 Snapshot cmd

断线重连必须要快照，不然必崩。
