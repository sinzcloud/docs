行，我给你一套**可直接落地的棋牌游戏后端协议设计**（HTTP + WebSocket），适合：大厅/房间/斗地主(ddz)/麻将(mj)，并支持断线重连。

# 一、整体通信架构（推荐）

## 1）HTTP 负责：账号 + 大厅数据 + 查询类

* 登录/注册
* 用户信息
* 配置
* 排行榜/战绩
* 支付

## 2）WebSocket 负责：实时状态同步

* 在线状态
* 匹配/房间
* 游戏对局（出牌、发牌、同步）
* 聊天/广播
* 心跳、断线重连

---

# 二、HTTP API 设计（REST）

统一前缀：

```http
/api/v1
```

统一响应格式：

```json
{
  "code": 0,
  "msg": "ok",
  "data": {}
}
```

---

## 1）认证模块 auth

### 登录

`POST /api/v1/auth/login`

```json
{ "account": "a", "password": "123" }
```

返回：

```json
{ "token": "xxx", "userId": 10001 }
```

### 刷新 token

`POST /api/v1/auth/refresh`

### 登出

`POST /api/v1/auth/logout`

---

## 2）用户模块 user

### 获取个人信息

`GET /api/v1/user/profile`

### 修改头像/昵称

`POST /api/v1/user/profile/update`

---

## 3）大厅模块 hall

### 获取大厅数据（一次性拉取）

`GET /api/v1/hall/init`
返回：

* 公告
* 活动
* 房间列表
* 匹配模式列表
* 推荐玩法

---

## 4）配置模块 config

### 拉取游戏配置

`GET /api/v1/config/game`
返回：

* ddz 规则
* mj 规则
* 房间倍率配置
* 道具配置

---

## 5）战绩模块 record

### 最近战绩

`GET /api/v1/record/list?page=1`

### 对局详情

`GET /api/v1/record/detail?id=xxx`

---

## 6）支付模块 pay

### 创建订单

`POST /api/v1/pay/order/create`

### 查询订单

`GET /api/v1/pay/order/query?id=xxx`

# 三、WebSocket 协议设计（核心）

连接地址：

```txt
ws://host/ws?token=xxx
```

消息格式统一（建议 JSON，后期可换 protobuf）：

```json
{
  "cmd": 1001,
  "seq": 1,
  "data": {}
}
```

服务器返回：

```json
{
  "cmd": 1001,
  "seq": 1,
  "code": 0,
  "msg": "ok",
  "data": {}
}
```

推送消息（seq=0）：

```json
{
  "cmd": 2001,
  "seq": 0,
  "data": {}
}
```

# 十一、断线重连（必须做）

服务器必须保存 Room Snapshot：

* 房间玩家列表
* 当前游戏状态（轮到谁、剩余牌墙、地主是谁）
* 手牌数量（别人的手牌只发 count）
* 当前出牌区
* 倍数/底分
* 当前倒计时剩余

重连时通过 `1002` 返回快照。

# 十二、客户端 Cocos 消息处理建议

客户端写一个统一 dispatcher：

```ts
onMessage(msg) {
  switch(msg.cmd) {
    case 9301: RoomController.onRoomUpdate(msg.data); break;
    case 2011: DdzController.onPlayCards(msg.data); break;
    case 3012: MjController.onActionHint(msg.data); break;
  }
}
```
