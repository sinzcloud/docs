# 1. Nacos 高频面试题

### 1）Nacos 能做什么？

* 服务注册与发现
* 配置中心
* 动态刷新配置
* 支持集群、健康检查

### 2）Nacos 和 Eureka 区别？

* Nacos 支持 **AP/CP 模式切换**，Eureka 主要偏 AP
* Nacos 自带配置中心，Eureka 不带
* Nacos 支持更丰富的服务管理功能

### 3）Nacos 的 CP/AP 是怎么切换的？

通过配置 `nacos.core.protocol` 或集群模式配置（面试常问思想：CAP权衡）

* **AP**：可用性优先，允许短暂不一致（注册中心常用）
* **CP**：一致性优先（配置中心更偏 CP）

### 4）Nacos 配置刷新原理？

* 客户端通过长轮询监听配置变化
* 配置变更后推送通知
* Spring Cloud 通过 `@RefreshScope` 或自动刷新机制更新 Bean

### 5）Nacos 配置隔离怎么做？

* `Namespace`：环境隔离（dev/test/prod）
* `Group`：项目组隔离
* `DataId`：具体配置文件

# 2. OpenFeign 高频面试题

### 1）Feign 是什么？和 RestTemplate 区别？

Feign 是声明式 HTTP 客户端，基于接口+注解调用服务。
相比 RestTemplate：

* Feign 更简洁
* 支持拦截器、熔断、负载均衡集成

### 2）Feign 底层是怎么调用的？

核心点：

* JDK 动态代理生成接口实现
* 底层 HTTP 客户端可能是：HttpURLConnection / OkHttp / Apache HttpClient

### 3）Feign 如何实现负载均衡？

通过 **Spring Cloud LoadBalancer**：

* 从注册中心获取实例列表
* 选择一个实例（轮询等策略）
* 发起请求

### 4）Feign 超时怎么设置？

常问配置项：

* `connectTimeout`
* `readTimeout`

### 5）Feign 如何传递 Token？

通过 `RequestInterceptor`：

* 在拦截器里把 JWT/Token 放到 Header

### 6）Feign 如何处理文件上传？

`@RequestPart MultipartFile file`

# 3. Gateway 高频面试题

### 1）Gateway 的核心功能是什么？

* 路由转发
* 统一鉴权
* 限流
* 跨域处理
* 灰度发布
* 统一日志

### 2）Gateway 和 Nginx 区别？

* Nginx：偏静态转发、负载均衡，性能极强
* Gateway：Java生态，适合动态路由、鉴权、业务逻辑扩展

面试常见回答：

> Nginx 做流量入口，Gateway 做业务网关。

### 3）Gateway 三大核心概念？

* **Route**：路由规则
* **Predicate**：匹配条件（Path、Method、Header等）
* **Filter**：过滤器（前置/后置逻辑）

### 4）Gateway Filter 分几类？

* GlobalFilter（全局）
* GatewayFilter（单路由）

### 5）Gateway 如何实现动态路由？

* 配置写在 Nacos
* Gateway 监听 Nacos 配置变更
* 动态刷新路由表

### 6）Gateway 如何实现鉴权？

常见做法：

* 在 GlobalFilter 中校验 Token/JWT
* 不通过则直接返回 401

# 4. Sentinel 高频面试题

### 1）Sentinel 是干什么的？

核心：**流量控制 + 熔断降级 + 系统保护**

### 2）Sentinel 限流算法是什么？

常问：

* QPS 限流
* 并发线程数限流
* 令牌桶/漏桶思想（Sentinel 内部实现基于滑动窗口统计）

### 3）Sentinel 熔断策略有哪些？

* 慢调用比例
* 异常比例
* 异常数

### 4）Sentinel 降级和熔断区别？

* **降级**：主动返回兜底数据（fallback）
* **熔断**：错误率高时切断调用，过一段时间半开恢复

### 5）Sentinel 和 Hystrix 区别？

* Hystrix 停止维护
* Sentinel 功能更丰富（限流、系统保护、热点参数）
* Sentinel 控制台可视化配置更方便

### 6）Sentinel 如何和 Gateway 集成？

通过 `spring-cloud-alibaba-sentinel-gateway`：

* 可以按路由维度限流
* 支持 API 分组限流

# 5. Seata 高频面试题

### 1）Seata 解决什么问题？

解决分布式事务一致性问题。

### 2）Seata 有哪些模式？

高频：

* **AT 模式**（最常用，基于 undo_log）
* **TCC 模式**（Try/Confirm/Cancel）
* **SAGA 模式**
* **XA 模式**

### 3）AT 模式原理是什么？

关键点：

* 一阶段：本地事务提交，同时记录 undo_log
* 二阶段：成功则删除 undo_log，失败则回滚（根据 undo_log 反向补偿）
* 需要依赖数据库支持（必须有主键）

### 4）Seata 的三大组件？

* **TC**（Transaction Coordinator）：事务协调者
* **TM**（Transaction Manager）：事务发起者
* **RM**（Resource Manager）：资源管理者（数据库）

### 5）Seata 如何保证隔离性？

通过全局锁（Global Lock），避免脏写。

### 6）Seata AT 模式缺点？

常问痛点：

* 性能开销较大（undo_log、全局锁）
* 不适合高并发热点更新
* 需要数据库支持（表结构要求严格）

# 6. SkyWalking 高频面试题

### 1）SkyWalking 能做什么？

* 分布式链路追踪（Trace）
* 性能监控（响应时间、吞吐）
* 服务拓扑图
* 日志关联 TraceId
* JVM 监控

### 2）SkyWalking 原理是什么？

核心：

* Java Agent 字节码增强（无侵入）
* 采集链路数据
* 上报到 OAP
* OAP 存储到 ES / H2 / MySQL（看配置）
* UI 展示拓扑和链路

### 3）TraceId 是怎么生成和传递的？

* Agent 在入口生成 TraceId
* 通过请求 Header 传递到下游服务
* 各服务 span 组成完整 trace

### 4）SkyWalking 和 Zipkin 区别？

* SkyWalking 偏 APM（监控能力强）
* Zipkin 偏链路追踪（轻量）

### 5）SkyWalking 怎么定位慢接口？

常见回答：

* 找某个服务的响应时间 TopN
* 点进 Trace
* 看哪个 span 耗时最长（DB、RPC、Redis）

# ⭐ 组合型高频面试题（最常考）

### 1）一次请求从 Gateway 到微服务，链路如何走？

* Gateway 接收请求 → Filter 鉴权/限流
* 通过 Nacos 发现服务实例
* LoadBalancer 选择实例
* Feign 调用下游
* Sentinel 做限流熔断
* SkyWalking 生成 trace/span
* Seata 如果开启全局事务，则 TC/TM/RM 协调提交或回滚

### 2）如何保证微服务高可用？

答题点：

* Gateway 限流保护入口
* Sentinel 熔断降级防雪崩
* Nacos 集群保证注册中心可用
* 服务多实例部署 + LoadBalancer
* Seata 控制分布式事务一致性
* SkyWalking 监控预警

### 3）服务雪崩是什么？怎么解决？

* 雪崩：某服务不可用导致调用链层层阻塞
  解决：
* 超时控制（Feign超时）
* 限流（Sentinel）
* 熔断降级（Sentinel fallback）
* 线程隔离（舱壁模式）
* 服务缓存/兜底返回

### 4）配置中心更新后怎么让服务无感刷新？

* Nacos 长轮询推送配置变更
* Spring 自动刷新或 `@RefreshScope`

### 5）Seata 全局事务失败会发生什么？

* TC 通知 RM 回滚
* undo_log 反向恢复数据
* 最终保证一致性

# ⭐ 面试必背“简答模板”（一句话版）

* **Nacos**：注册发现 + 配置中心，支持 AP/CP。
* **Feign**：声明式 HTTP 调用，动态代理生成客户端。
* **Gateway**：基于 WebFlux 的网关，Route + Predicate + Filter。
* **Sentinel**：限流熔断降级，保护系统防雪崩。
* **Seata**：分布式事务框架，AT/TCC/SAGA/XA。
* **SkyWalking**：无侵入 APM，通过 Java Agent 采集 Trace。
