“高并发项目”本质就是：**在高 QPS/高连接数情况下，系统依然稳定、响应快、不崩溃**。实现思路不是靠某一个技术，而是靠一整套架构和工程手段。

下面给你一个比较标准的实现路线（偏实战）。

---

## 1. 架构核心：拆分 + 横向扩展

高并发系统一定要做到：

* **无状态服务**（服务本身不存 session）
* 服务可以**水平扩容**（加机器就能抗更多请求）
* 用 **负载均衡** 分摊流量（Nginx / SLB / Envoy）

典型结构：

用户请求 → CDN → Nginx/LB → 应用集群 → 缓存Redis → 数据库MySQL

---

## 2. 高并发第一招：缓存体系（Redis）

大部分并发问题，本质是数据库扛不住。

所以必须用 Redis 做：

* 热点数据缓存（读多写少）
* session/token存储
* 分布式锁
* 计数器、排行榜
* 限流（令牌桶/滑动窗口）

### 常见策略

* Cache Aside（旁路缓存）：先查缓存，没有再查 DB 并写缓存
* 热点数据提前预热
* 缓存过期随机化，避免雪崩

---

## 3. 数据库抗压：分库分表 + 读写分离

数据库是瓶颈核心。

### 常用优化方式

* **索引设计**（必须）
* SQL优化（避免大表全扫）
* **读写分离**（主写从读）
* **分库分表**（按 user_id/order_id hash）

并发很高的业务（订单、交易）通常都会做分库分表。

---

## 4. 异步化：消息队列削峰填谷

高并发场景很多请求其实不需要同步完成，比如：

* 下单后发短信
* 写日志
* 生成报表
* 积分计算

用 MQ（Kafka / RocketMQ / RabbitMQ）：

* 请求先快速返回
* 业务慢慢消费处理
* 可以抗流量峰值

这叫 **削峰填谷**。

---

## 5. 限流 + 熔断 + 降级（必须）

没有限流的系统，流量一大就会直接炸。

### 限流

* Nginx 限流（IP级别、接口级别）
* Redis 限流（分布式）
* 网关限流（Spring Cloud Gateway）

算法：

* 令牌桶
* 漏桶
* 滑动窗口

### 熔断/降级

* 某个下游服务慢 → 直接熔断
* 返回兜底数据（默认值、缓存数据）

常用框架：Sentinel / Resilience4j

---

## 6. 分布式锁与一致性控制

高并发最典型的问题：

* 超卖（库存扣减）
* 重复支付
* 重复下单

解决方式：

* Redis 分布式锁（SETNX + 过期）
* 数据库乐观锁（version字段）
* 幂等设计（请求唯一ID）

**高并发系统一定要支持幂等**，否则重复请求会导致数据错乱。

---

## 7. 秒杀/抢购类：热点隔离 + 库存方案

秒杀是高并发最难的经典案例。

常用方案：

### 方案A：Redis预扣库存（推荐）

* 库存提前放 Redis
* 请求到来先原子扣减 Redis
* 成功后异步写 MySQL（MQ消费落库）

Redis 原子操作（decr）保证高性能。

### 方案B：队列排队

用户请求 → 进入队列 → 逐个消费处理
保证数据库不会被瞬间打爆。

---

## 8. 静态化 + CDN（提升巨大）

很多高并发系统，真正的关键是：

* **尽量不要打到后端**
* 静态资源直接 CDN 返回

比如商品详情页：

* 静态 HTML 缓存
* 页面模板渲染后缓存
* 只请求少量动态接口

---

## 9. 高性能网络模型（服务层）

应用服务器本身要能抗连接：

* Java：Netty / Spring WebFlux
* Go：天然高并发协程
* Node：事件驱动模型

并发高时要注意：

* 线程池配置
* 数据库连接池配置
* 避免阻塞IO（比如大文件读写）

---

## 10. 监控 + 压测 + 自动扩容

高并发系统如果没有监控就是“盲人开车”。

必须具备：

* Prometheus + Grafana（指标监控）
* ELK / Loki（日志）
* 链路追踪：SkyWalking / Jaeger
* 压测工具：JMeter / Gatling / Locust
* 自动扩容：K8s HPA

---

# 一个标准高并发项目落地方案（参考架构）

### 请求链路

CDN → Nginx → 网关限流 → 应用集群

### 应用层

* 无状态服务
* Redis缓存热点数据
* MQ异步处理耗时任务

### 数据层

* MySQL主从 + 分库分表
* Redis热点 + 计数器
* ES做搜索
* 对账/报表用离线任务

---

# 总结一句话

高并发项目靠的是：

**缓存抗读 + MQ抗写 + 分库分表抗数据 + 限流熔断抗突发 + 横向扩容抗增长**
