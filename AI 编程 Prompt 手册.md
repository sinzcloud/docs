# AI 编程 Prompt 手册（工程师必备）

> 本手册整理了一套高复用、高命中率的 AI Prompt 模板，适用于 Claude / ChatGPT / GPT-4 等大模型。  
> 目标：让 AI 输出更接近 **生产可用代码**，减少废话，提高开发效率。

---

## 1. Prompt 使用原则（必看）

### 1.1 让 AI 输出更像工程师的关键
每次提问尽量包含：

- **背景**（你要做什么项目）
- **目标**（你希望实现什么功能）
- **输入/输出**（参数、返回值）
- **约束条件**（技术栈、数据库、框架、规范）
- **输出格式**（让 AI 按章节输出）
- **质量要求**（生产级、可运行、带异常处理）

---

### 1.2 推荐万能补充句（强烈建议加）
```text
请以生产可用的标准输出，不要只给概念，必须给出可落地的代码、目录结构和示例。
```

## 2. 通用万能 Prompt（适用任何场景）

```text
你是一个资深软件架构师 + 高级工程师。

背景：
我正在做一个【项目类型】项目，技术栈是【技术栈】。
业务目标是：【一句话描述目标】。

需求：
【详细描述需求/问题】

约束：
- 代码风格：清晰、可维护
- 需要考虑异常处理、边界条件
- 输出必须包含可直接运行的示例

输出格式：
1. 方案说明
2. 核心设计
3. 关键代码
4. 测试建议
5. 注意事项
```

## 14. 最佳实践：Prompt 组合用法

### 14.1 推荐提问顺序
开发一个完整模块时，建议这样问 AI：

1. **架构设计 Prompt**（定模块）
2. **DDD Prompt**（定领域模型）
3. **数据库 Prompt**（定表结构索引）
4. **Spring Boot Prompt**（写接口代码）
5. **Vue Prompt**（写前端页面）
6. **JMeter Prompt**（压测验证）
7. **README Prompt**（包装项目）

## 15. Prompt 示例（可直接复制）

### 示例：跨境电商 ERP 订单模块
```text
你是资深 Java 架构师，精通 Spring Boot、MySQL、Redis、DDD。

背景：
我要开发跨境电商 ERP 系统的订单模块（Order Module）。

需求：
- 创建订单
- 查询订单列表（分页）
- 订单取消
- 订单状态流转（NEW -> PAID -> SHIPPED -> FINISHED）

约束：
- 采用 DDD 分层架构
- MySQL 存储订单
- Redis 做缓存
- 接口要幂等，防止重复下单

请输出：
1. 数据库表结构（订单表、订单项表）
2. DDD 分层代码结构
3. 核心代码（Controller/Service/Domain）
4. 状态机设计
5. 单元测试示例
```
